# ProChefDesk — Sürüm geçmişi

Mevcut sürüm: **v2.6.95**

Yapısal kilometre taşları (kronolojik tersine):

* **v2.6.95** — Faz 5 Adım 2: Cloud sync queue persistence. Önceki davranış: `cloud-pertable.queue` yalnızca JS hafızasında. Offline yazımda (Wi-Fi kesik mutfakta veya mobil bağlantı kötü iken) kullanıcı sekme kapatırsa kuyruk uçuyordu — cloud'a hiç gitmemiş yazımlar diğer cihazlarda görünmüyordu, hatta lokal IDB'de var olsa bile pull/sync döngüsünde çelişki yaratabiliyordu. Yeni davranış: her `queueUpsert`/`queueDelete` mutation'ı sonrası queue snapshot'ı 250ms debounce ile IDB'ye yazılıyor (key: `pertable_queue`). Boot sırasında `_loadPersistedQueue()` async tetikleniyor — kalan kuyruk varsa queue array'ine merge ediliyor, online + ready ise `scheduleFlush` çağrılıyor. `flushNow` Promise.all sonrası queue mevcut durumunu IDB'ye yansıtıyor (flush sırasında gelen yeni mutation'lar korunur). `wipeAllUserData` IDB queue'yu da temizliyor (Clear all sonrası phantom item'lar dönmesin diye). Race koruması: load sırasında canlı queue'da aynı `dedupeKey` varsa persisted item atlanıyor (yeni canlı kayıt daha güncel). Idempotent: upsert'ler `onConflict` ile, duplicate yazım güvenli. Etki: tarayıcı kapansa, cihaz uyusa, ağ kesilse — yazılan veri reload sonrası otomatik tekrar gönderilir.
* **v2.6.94** — Demo cleanup: Account ekranındaki "Re-add/Remove demo recipes" butonu tamamen kaldırıldı. Operatör kararı: kullanıcı için demo Add/Remove opsiyonu yok; misafir siteye girdiğinde tek seferlik seed olur (Fix #2 user-bağımlı gate sayesinde sign-in sonrası asla tetiklenmez), sign-in olunca `auth.js`'in `clearUserData` çağrısı tüm demo'yu temizler. Yapılan: (1) `account.js`'ten `demoToggleBtn` HTML ve click handler silindi; (2) `demo-recipes.js`'ten `removeDemo()` fonksiyonu kaldırıldı, public API artık `PCD.demo = { seed }`; (3) i18n string'leri (`clear_demo`, `reset_demo`, `demo_3_recipes`) 6 dil dosyasından (en, tr, es, fr, de, ar) silindi — Türkçe çeviriler dahil. Sıfır kalıntı referansı.
* **v2.6.93** — Faz 5 Adım 1: Misafir/üye demo ayrımı + welcome tour cloud-bağımsızlığı + backup restore tam zincir + Clear all cloud temizliği. **Dört fix tek pakette, birbirini destekliyor:** (1) `app.js`'in demo seed gate'i artık user-bağımlı (`if (!user && !demoSeeded)`) — sign-in olmuş kullanıcı için demo asla otomatik tetiklenmez; misafir ilk açılışta bir kez seed olur. Önceki davranışta her F5'te boot pull → cloud user_prefs.onboarding boş → demoSeeded=false → demo recipe duplicate ediliyordu (9→12→15 progression). (2) Welcome tour bayrağı state'ten localStorage'a taşındı (`pcd_tour_done`); `state.onboarding.mainTourDone` cloud-pertable sync'inde güvenilir yazılmıyordu, her F5'te tour geri açılıyordu. localStorage cloud-bağımsız, F5/sign-in/out'tan etkilenmez. State flag'i fallback olarak okunuyor (eski cihazlardan true gelen kullanıcılar için tour açılmasın). (3) Backup restore handler async/await zinciri: `wipeAllUserData` (RLS user_id ile 19 tabloda DELETE) → `replaceAll` (runMigrations + orphan ws cleanup + activeWorkspaceId validation + `flushSync` await) → `queueFullState` → `flushNow` await → reload. Önceki handler `store.set()` debounced + 800ms timeout idi, hiçbir adım garantili tamamlanmıyordu, cloud da temizlenmiyordu. (4) Clear all data handler artık `wipeAllUserData` çağırıyor; önceki davranış `cloud.pushNow()` idi ama bu v2.6.87'den beri no-op — Clear all aslında cloud'u hiç temizlemiyordu, reload sonrası pull her şeyi geri çekiyordu. **Yeni public API:** `PCD.cloudPerTable.wipeAllUserData()`, `PCD.cloudPerTable.queueFullState()`. `store.flushSync` ve `cloud-pertable.flushNow` artık Promise döndürür (eski sync caller'lar Promise'i discard ediyor, davranış aynı).
* **v2.6.92** — Faz 4 Adım 4c: localStorage yazma yolu kapatıldı + tek seferlik LS cleanup. `store.js`'e iki module-level flag eklendi (`\_idbWriteOnly`, `\_migrationDone`). İlk başarılı IDB persist'inden sonra `\_completeMigration()` çağrılır: `state.prefs.idbWriteOnly = true` (kalıcı flag), `localStorage.removeItem(LS\_KEY\_STATE)`, modül flag'leri set. Boot'ta `load()` flag'i okur ve `\_idbWriteOnly` aktif eder. `persist`, `flushSync`, `trimAndPersist` artık `\_idbWriteOnly === true` iken LS yazımını atlar — sadece IDB'ye yazar. `reset()` sign-out döngüsünde flag'leri sıfırlar. Sonuç: localStorage 5 MB sınırı fiilen kaldırıldı; tek state kaynağı IDB. **Faz 4 tamamlandı.**
* **v2.6.91** — Faz 4 Adım 4b: Okuma yolu IndexedDB'ye geçti. `store.js` `load()` async oldu — önce IDB'den (`prochefdesk` DB, `state` store, `main` key) okuyor, IDB boşsa `localStorage` fallback'i. `store.init()` Promise döndürür. `app.js` `boot()` async function olarak hem `await PCD.store.init()` hem `await PCD.auth.init()` ediyor — auth zinciri (getSession → cloud.pull → photo migrate → cloud-migrate) tamamlanmadan `onAuthResolved()` çağrılmıyor, router register pull'dan SONRA, UI listener'lar pull-edilmiş state üzerine kuruluyor. Boot'un her adımına teşhis için `console.log('\[boot] ...')` eklendi.
* **v2.6.89** — Faz 4 Adım 4a: IndexedDB altyapısı + write-through. Yeni dosya `js/core/idb-wrapper.js` (\~110 satır, native IndexedDB üzerine Promise wrapper). `store.js`'in 4 yazma noktası (`persist`, `flushSync`, `trimAndPersist`, `reset`) artık LS'ye yazdıktan sonra IDB'ye de paralel yazıyor (fire-and-forget). Okuma hâlâ LS'den. Migration script gereksiz: write-through pattern'inde ilk persist çağrısı IDB'yi otomatik dolduruyor.
* **v2.6.88 (schema-only)** — `cost\_history` tablosu drop edildi. v2.6.71'de Faz 4 hazırlığı sırasında eklenmişti ama hiç yazılmadı; costHistory verisi `user\_prefs.data.costHistory` jsonb içinde yaşıyor. Migration: `migrations/v2.6.88-drop-cost-history.sql`. Kod tarafında değişiklik yok.
